Machine Learning Data Enrichment

Overview

Este script define un sistema para la detección y enriquecimiento de anomalías en registros de autenticación de un servidor. La detección se realiza usando un modelo de machine learning que identifica comportamientos atípicos en logs de autenticación, y el enriquecimiento de estos datos se realiza a través de consultas a servicios externos como VirusTotal y AbuseIPDB, proporcionando contexto adicional sobre las IPs identificadas.

Process

1. Inicialización de la clase

Esta clase se encarga de configurar el modelo de detección de anomalías y de definir el archivo de log a analizar. Se define un modelo IsolationForest con una tasa de "contaminación" del 10%, que controla cuántos datos son esperados como anomalías. También configura el logger para registrar el proceso.

2. Parsea líneas de log con parse_log_line:

Esta función toma cada línea del log y utiliza una expresión regular para extraer información de eventos fallidos, exitosos, y su dirección IP. Si la línea no coincide con el patrón esperado, se registra el error.

3. Extrae características numéricas de los logs en extract_features:

A partir de los registros parseados, se generan características como el número de eventos, el ratio de fallos y el número de eventos únicos por IP. Estas características serán la base para que el modelo detecte anomalías.

4. Entrena el modelo y detecta anomalías con train_and_detect:

Se cargan los registros, se normalizan las características extraídas, y se aplican al modelo IsolationForest. Las IPs detectadas como anomalías se almacenan en un reporte que muestra sus detalles.

5. Enriquece anomalías con información externa:

La clase LogEnrichment se conecta a APIs externas para obtener geolocalización, reputación, y amenazas activas de cada IP en la lista de anomalías, agregando contexto que puede ser útil para el análisis de seguridad.
